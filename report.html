Client Critique

The page overall did a very good job at depicting the band and being easy to navigate. It had all the features of a website I was expecting and was organized very well. The images and videos were from all of our major performances giving the viewer a wide perspective to see. There was a good balance of humor as well as intention to be personable to the viewer. The contact and book us pages are a little lacking in terms of content. The photos page could be organized in a more meaningful way, because right now it is just a bunch of random photos. The color scheme and theme were subdued and appealing. Certainly not distracting but they could be modernized a bit. --Alex Mazur (of CJ & the Boys)

Itemized Assignment Expectation Report

Your site must meet the following expectations:

  1. You will create a website for a client of your choosing. Your client could be a family member, a business, a student group, a friend, an online community, and so on. The only restriction is that the client cannot be just you.
    • This website was created for the brass band "CJ & the Boys", in which I am a member. However, I am not the only member, as there are 6 original core members.
  2. Your site must include some sort of header.
    • This website contains a header at the top of each page. It contains the band name and a navigation bar.
    • This header also sets the theme colors for the site, and all other colors in the site are usually based off of these two colors.
  3. Your site must contain at least five separate pages, each having a meaningful purpose.
    • Home (index) -- introduces the viewer to the band and provides a concise mission statement of the band.
    • About -- expands on who we are as a band, introducing the individuals and providing history of the band as a whole.
    • Videos -- is full of many embedded YouTube videos of previous performances.
    • Photos -- contains various miscellaneous photos of CJ & the Boys with the intent of making the band more personable. (However, many of the URL's to the images that I had were mysteriously removed, so most photos are currently duplicates on the photos on other pages of the site.)
    • Contact -- shows the user an embedded facebook window to stay in contact with the band, and provides other means of getting in contact.
    • Booking -- provides a fillable form that asks for details regarding a requested gig from the band.
  4. Your site must include some sort of navigation links.
    • The navbar at the top of each page contains navigation links to the other pages and shows which page you are currently on.
  5. Your site must support distinct layouts for large and small screens. For instance, multicolumn layouts on large screens should be reduced to single column layouts on small screens. You are encouraged to implement separate layouts using CSS media queries.
    • This website uses lots of flexbox layouts for pictures or video cards, so they resize with the window when it grows or shrinks.
    • I did not sufficiently make the website responsive, and so some things would not appear clean on a mobile device such as the navigation bar. That is something I will work on in the future.
  6. Your site must maintain a coherent style across its pages. Global style rules must be placed in a shared stylesheet. Individual pages’ style rules must be placed in a separate stylesheets.
    • There is a global stylesheet.css that is applied to each page on the site, and each page has its own style sheet in the same css directory.
  7. Your site must include images, and their alt attributes must adequately describe the image content to users of screen readers, user with images disabled, and webcrawlers.
    • There are many images in this site, all of either the whole band or the members involved.
    • All images in the site are labelled with appropriate alt attributes.
  8. Each page must declare its favicon.
    • Each page on this site declares a custom-made favicon (which I made on https://www.favicon.cc)
    • Note that the same favicon is used on every page in the site, not a unique favicon for each page.
  9. Your site's non-HTML resources must be organized. Images must be placed in a directory named images. CSS files must be placed in a directory named css. JavaScript must be placed in a directory named js.
    • I created an images folder, but I couldn't figure out how to have images in this directory and use them in other pages, so it's currently blank.
    • All CSS files are in a directory names css, and there exists an images directory.
  10. Your site must not be built using any libraries or frameworks. Do not use Bootstrap, jQuery, React, Angular, Vue, or any other third-party tools. You may, however, reimplement the ideas you see in these third-party tools.
    • I did not use any libraries or frameworks in the building of this site.
  11. Each page and stylesheet of your site must validate without any warnings or errors using the W3C’s HTML validator and CSS validator.
    • All pages except the Videos and Contact page pass the HTML validator, and these only give errors because of the iFrame elements within. However, I chose to embed YouTube videos directly on my site, and I trust their embedding iFrame process more than my CSS skills, so I decided to leave the iFrames intact.
  12. Your site must be stored in a Git repository that you have shared with your instructor. On GitHub, GitLab, and Bitbucket, share with the user twodee.
    • This site is stored in the Git repository Project1 by user arnebesp2531, which was shared with Dr. Johnson via Slack message.
    • I couldn't figure out how to share a Git repository with another user, so I'm really hoping that isn't a critical part of the process.
  13. The names of your files should be entirely lowercase and contain only alphanumeric characters.
    • This is true of all files in this site.
  14. Your site must be served out of port 8443.
    • The website is served out of port 8443.
  15. Your site must be accessed only through HTTPS. You are encouraged to use Let’s Encrypt to obtain a free certificate.
    • The suggested steps toward obtaining a certificate were used, and the site is only accessible through HTTPS.
  16. Your content should be neatly aligned. The axes should be clearly visible.
    • The videos page consists of columns that have very strong axes of alignment, both vertically and horizontally.
    • The other pages either have a similar structure or are centered on the page intentionally.
  17. Your site must use at least two different font faces. Headings are often rendered in a sans-serif font, and copy in a serif font.
    • The headings on the site are set to sans-serif, and everything else is serif (or default).
  18. Your site must include at least one form (or informal collection of input elements). The form might submit data to a server or invoke some client-side JavaScript function that alters the page. The action is not specified.
    • The Booking page of this site contains a form that would gather information pertinent to a potential gig for the band. Although it does not have a specified action yet, this form would be used to create a notification for the members of CJ&tB to let them know to follow up with the potential customer.
  19. Your site must include links to external sites. Clicking on external links must open a new tab or window rather than replace your page.
    • The About page includes links to https://www.luckychops.com/ and https://www.youngbloodbrassband.com/, and it opens in a new tab when they click the anchor link.
  20. Your source code must be readable, using conventional and consistent formatting and meaningful names for classes and IDs.
    • In the html and CSS code, I believe that I used conventional formatting. I names classes and IDs in a way that I would understand what they were upon seeing them briefly.
  21. Your site must include at least one animation, powered either by CSS or JavaScript.
    • I used CSS to create a hover color-changing effect when the user is hovering above the navigation bar. When they go away, the color returns back to the normal color of the button.
